# HelloDG

把「DG 视角顾问」做成一个可复用的 Codex Skill。

## 这是什么

- **DG 视角顾问**：偏“决策优先”的政企数字化项目顾问风格，兼顾产品方案与正式写作；也能聊生活情绪但不空泛。
- Skill 定义文件在：`.agents/skills/dg/SKILL.md`

## 适用场景

- 政企数字化项目：范围收口、一期边界、方案写作、风险与验收口径
- 沟通表达：把话讲明白、把争议点掰开揉碎、把决策拉回可执行
- 生活唠嗑：情绪/选择题/纠结（但不“鸡汤式”说教）

## 怎么用

在 Codex 项目里放好这个目录结构后，对话里用以下任一触发词唤醒：

- `DG` / `DG顾问` / `DG视角` / `你好DG`
- `收口` / `一期边界` / `给我终稿` / `更政府一点` / `别AI味`
- `给用户讲得明白 + 给技术列清单`

退出扮演：

- `退出DG`（回到普通助手口吻）

## 默认全量装载（不想每次唤醒）

如果你希望在打开这个仓库时 **DG 默认常驻**（包含“动作 + 口癖”），直接用本仓库根目录的 `AGENTS.md`。

- 默认：DG 常驻（无需触发词）
- 结构化开关：`收口/拍板/给我清单/按DG流程/写成可交付/给我终稿`
- 退出：`退出DG/切回正常`

## 交互规则（简版）

- 默认是“像真人唠嗑”的输出；你说 `收口/拍板/给清单/给终稿` 才切到更结构化、更硬的交付稿。
- 你可以要求口吻（例如“更政府一点/更正式/别AI味/带点口音”），DG 会按要求收敛或放开。
- 约定：当你叫 `DG/你好DG` 时，默认进入“完整版 DG”（可以带（动作）+ 口癖）；你说 `收敛/别演/正式一点` 就恢复收敛版。

## 仓库结构

- `SKILL.md`：仓库对外入口（给 GitHub 读者看的简介版）
- `.agents/skills/dg/SKILL.md`：Codex 使用的 DG skill 主入口与行为规范（核心文件）
- `examples/`：效果示例（建议用真实问题/对话展示“唠嗑→收口”的过程）
- `references/research/`：方法论与材料沉淀（术语口径、模板、打法，支撑 SKILL 的可迭代）
- `LICENSE`：MIT License

## 贡献方式

欢迎用 PR 的方式补充：

- 新增 `examples/*.md`（一个问题一个示例，越贴近真实越好）
- 在 `references/research/` 里沉淀“可复用模板/口径/打法”
